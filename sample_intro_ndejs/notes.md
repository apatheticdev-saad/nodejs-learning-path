# 📘 Introduction to Node.js

## 🔹 Why Backend is Needed

In web development, there are two main parts:

* **Frontend** → What users see (UI, buttons, forms)

* **Backend** → Handles logic and data behind the scenes

### Backend Responsibilities:

* Storing & retrieving data 📦
* User authentication (login/signup) 🔐
* Business logic processing ⚙️
* Handling API requests 🌐

👉 Without backend, websites remain **static** and cannot store or process data.

---

## 🔹 What is Node.js?

Node.js is a **runtime environment** that allows JavaScript to run on the **server-side**.

👉 Normally, JavaScript runs only in the browser
👉 With Node.js, it can run on server also

---

## 🔹 Key Features

* Built on **Chrome V8 Engine** ⚡
* Non-blocking (asynchronous) architecture
* Handles multiple requests efficiently
* Large ecosystem via **npm**  

---

## 🔹 Client-side vs Server-side JS

| Feature | Client-side      | Server-side (Node.js) |
| ------- | ---------------- | --------------------- |
| Runs on | Browser          | Server                |
| Purpose | UI & interaction | Logic & data handling |
| Access  | Limited          | Full system access    |

---

## 🔹 Summary

* Backend is essential for dynamic apps  
* Node.js enables backend using JavaScript
* Makes full-stack development easier 🚀
